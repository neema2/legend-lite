  1. Relational synth returns Class[*] (unchanged). M2M unchanged.
  2. Class-typed Join PMs NO LONGER inline-materialize ^Firm(...) from sibling
     mapping. They emit bare $row.<alias> — sub-row in class-typed slot. Type

  3. Associate alias = full join-chain name joined by '__' (Position 2).
     Single hop: 'Person_Firm'. Multi-hop: 'Person_Address__Address_City'.

     per unique chain prefix in the synth).

     No more enumValue(...) native (was never registered).
  5. JoinMediated filter uses the same alias space as PMs — no '__filter' magic.
  6. PM-name validation: must match a declared property; must not collide with
     any associate alias the synth emits.

KEY FINDING — first()/head() are NOT Pure Relation functions:
  Pure has: first<T>(T[*]):T[0..1] (collection-only), first(rel,window,row)
  (window-context only), and limit<T>(rel:Relation,n:Int):Relation<T> for
  relations. There is NO first(Relation):Relation overload in
  Pure.java#FIRST_*. Legacy plangen at plangen-legacy-pre-port.java.txt:480
  had a non-spec hack (case "first" -> generateFirst -> LIMIT 1).
  Tests like SlicingCheckerTest.testFirst exercise this hack
  (#TDS#->sort()->first()); they were broken in baseline AND remain broken.
  NOT Phase A's responsibility.

  TypedGetAll:1.
