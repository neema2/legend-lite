# Core module audit — 2026-07 (four parallel auditors + prior lowering audit)

Full-module sweep: parser/lexer, F layer + catalog + normalizer, G layer,
cross-cutting architecture. The lowering/sql/exec audit ran earlier (its
HIGHs are fixed; remainder in DESIGN_DEBT.md). Severity: H / M / L.
This file is the module-wide ledger; strike items as they land.

## TIER 1 — correctness holes and lies (fix first)

1. **[H][F] `findProperty` breaks its own contract — association navigation
   dead.** `ModelContext.findProperty` promises "association-injected
   navigation properties"; `PureModelContext:116-133` walks only declared
   properties + supers, never the association index. Every G checker routes
   through it, so `$person.firm` via an Association fails "no property".
   Likely a large slice of the corpus H-bucket ("has no property" errors).
   Fix: add the association-index arm (mirror `ModelBuilder.findAssociationProperty`).
2. **[H][G] `distinct(~col)` corpus bug diagnosed.** Real Pure registers
   only the ColSpecArray overload (verified in real checkout). Fix in
   `DistinctChecker`: desugar single ColSpec -> one-element ColSpecArray
   (the ProjectChecker/SortChecker pattern). Do NOT add a fake overload.
3. **[H][G] `FlattenChecker` bypasses its registered signature entirely** —
   the `tableReference`-went-vestigial disease repeated. `FLATTEN__T_MANY__
   COL_SPEC_1` is dead. Route call validation through the signature; keep
   the bespoke output schema. Partial offenders: GraphFetchChecker (magic
   string "serialize", blind `sig.get(0)`), MatchChecker (doc claims
   signature governs; code hardcodes), EvalChecker (dead registrations).
4. **[H][arch] Sealed-switch `default ->` arms disable the module's chosen
   safety mechanism** — root package-info forbids them; 58 exist, including
   `Typer:138` over sealed `ValueSpecification` itself and `PureSql:27`.
   Sweep sealed-type switches; scope the invariant; note it's tool-unenforced.
5. **[H][arch] `isMany` x5 with DIVERGENT `Multiplicity.Var` semantics**
   (checkers: var != many; exec/lowering: var = many). Possibly live bug.
   Fix: `Multiplicity.isMany()` once; write down the Var decision.
6. **[H][parser] JsonModelConnection islands silently dropped**
   (`ElementParser:995-1002`): non-matching embedded connections are
   consumed and discarded with zero diagnostics. Throw positioned errors.
7. **[M-H][G] Kernel disagrees with itself:** overload SCORING accepts class
   subtypes (`paramTypeScore`), UNIFY is identity-only — resolution can pick
   a winner unify rejects, yielding "expected Person, got Employee".
   Align both (identity or subtype) until requireConforms lands.
8. **[M-H][G] Missing multiplicity checks:** IfChecker (condition mult),
   EvalChecker.variableEval, NewChecker (`[2]` into `[1]` property passes).
   Silent DESC fallback in SortChecker (any non-"DESC" enum -> ASC, FQN
   never validated). `commonSupertype` widens mismatched relations to Any.
9. **[M-H][F] No inheritance-cycle guard in Phase F** — cyclic `extends`
   stack-overflows on first member lookup. Validate acyclicity in
   ModelIntegrity (one DFS).
10. **[M-H][G] `TypedNewInstance` uses `Map.copyOf`** — destroys property
    order; HIR child order is JVM-run-nondeterministic. Fix before any
    lowering consumer (LinkedHashMap wrap or entry list).

## TIER 2 — architecture (the structural debts)

11. **[H][arch] Error taxonomy.** User error / internal bug / unimplemented
    indistinguishable by type past the parser (ISE=115, TIE=118, IAE=66,
    UOE=14; PipelineStageFailureTest greps class-name strings — the smoking
    gun). Proposed: `LegendCompileException` base (phase + optional pos)
    with ParseException/ResolutionException/ModelException/
    TypeInferenceException/LoweringException subtypes; NotImplementedException
    for backlog; ISE reserved for genuine invariants. Migrate ~50 obvious
    sites first.
12. **[H][arch] Source positions die at the parser boundary.** Lexer has
    offsets; parser errors have [line:col]; no AST record carries a span.
    Path: (a) stopgap — SpecCompiler/Typer wrap errors with enclosing
    function FQN; (b) real — `SourceSpan(startOffset, endOffset)` on ~12
    top-level element records + ValueSpecification (~100 construction
    sites); offsets only, line/col lazily; (c) shared [line:col] formatter.
13. **[H][arch] Two package cycles, confirmed:**
    - compiler<->normalizer: one direction FQN-hidden
      (`ModelBuilder.from(NormalizedModel)` at ModelBuilder:261). Break:
      delete that overload (callers unwrap); then re-home ModelBuilder
      (it's a parsed-model INDEX, misnamed and mis-homed) + SynthFqn.
    - spec<->spec.typed: entirely `ExprType`'s location (63 reverse refs,
      all one import). Move ExprType; mechanical ~65-file rename.
    Then add the ArchUnit `layeredArchitecture` rule so neither regresses —
    the guardrail that would have caught all of this mechanically.
14. **[M-H][arch] Parser AST leaks into HIR/backend:** TypedCDate/TypedCTime
    carry parser literals (Lowerer pattern-matches parser types); lowering
    dispatch keyed on `IdentityHashMap<parser.Function, Rule>`. Fix: typed-
    layer date/time value model; dispatch on a stable id (CoreFn-style).
15. **[H][F] Bare-name catalog silent-capture** — user fn at `meta::pure::
    …::map` silently rebinds to the native. Full FQN-migration map is in
    the F-audit (7 layers, ordered: catalog+resolver together first,
    CoreFn.of is the choke point, then emission sites). Interim: reject
    user definitions under `meta::pure::` or check collision loudly.
16. **[H][E] MappingNormalizer god class (2,166 lines, 12 jobs).** Four
    extraction seams: LegacyMappingPrepass (extends/multi-hop/M2M-cycles),
    RelOpTranslator (nullable-mode params -> sealed TranslationCtx),
    JoinChainEmitter (11-param method -> HopContext), ViewExpander.
    Also: view-filter + `~distinct` post-hoc AST surgery likely BROKEN
    today (layerMappingFilterPreMap expects bare map(...) terminal;
    ~distinct wraps it) — test and fix by threading filters as arguments.
17. **[M][F] ModelBuilder last-write-wins on duplicate FQNs** (documented
    but stale rationale); runtime cross-bake silently skips unknown
    mappings; TypeClassifier Generic-head names unvalidated
    (`Bogus<T>` classifies fine).

## TIER 3 — parser sweep

18. **[H] Stringly dispatch despite minted tokens:** service/runtime/
    connection/database body keys re-read as strings while dedicated
    TokenTypes sit unused (6+ sites enumerated in the parser audit).
    Dispatch on token types; delete `LogicalOp.fromKeyword`.
19. **[H] Dead token surface:** 12 fully dead tokens + ~71 keyword tokens
    alive only as IDENTIFIER_TOKENS passthrough (each one a two-place-
    update trap). Delete or dispatch.
20. **[M] Join-chain grammar TRIPLICATED with capability drift** (view
    ~filter path lacks per-hop join types). One parseJoinChain.
    Function-signature grammar duplicated (parseFunctionDefinition vs
    parseNativeFunction); brace-block scan x3.
21. **[M] SpecParser bypasses its own cursor** — 89 raw `tokens.type(pos)`/
    `pos++` sites next to inherited peek()/match()/advance(). Mechanical.
22. **[M] Lexer correctness:** `%...T10:30Z` unlexable (Z suffix; TZ block
    dead code); leading-dot floats `.5` unlexable (verify vs real grammar
    first); unterminated strings/comments accepted to EOF.
23. **[M] Quoted identifiers keep quotes+escapes** in TSC.parseIdentifier
    and `$'my var'` (let-name vs reference disagree!); unquote logic
    duplicated 8x, EP never unescapes. One shared unquoteAndUnescape.
24. **[M] `#>{db.schema.TABLE}#` mis-split** (lastIndexOf('.')) — schema
    refs produce structurally invalid FQNs. Nested islands truncate at
    first ISLAND_END; island re-lex loses positions; graph-fetch trailing
    garbage ignored. Service parser permissive (execution flavor accepts
    anything; autoActivateUpdates accepts any token; missing pattern
    defaults "/"). ImportScope silently discards non-qualified imports.
    IDENTIFIER_TOKENS is a MUTABLE public EnumSet (wrap unmodifiable).
    ModelIndexer rejects `native function` (ElementKind.NATIVE->CLASS lie).
25. **[L] Error-style: bare `error(...)` statements vs `throw error(...)`;
    raw NumberFormatException/IAE escaping (RelationalDataType, JoinType,
    parseInt sites); default expect() messages print raw enum names;
    trailing-comma tolerance inconsistent (verify each vs engine).
