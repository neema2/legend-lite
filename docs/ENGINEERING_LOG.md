## What this project is

legend-lite: a clean-room, backwards-compatible reimplementation of FINOS
legend-engine — parse → normalize → type-check (G) → resolve (H) → lower (I)
→ render (J) → execute (K) on DuckDB — verified against the REAL engine's
core_relational corpus (2497 tests). **Row equality with engine semantics is
the contract; golden SQL text is advisory.** Local reference checkouts:
`/Users/neema/legend/legend-engine` and `/Users/neema/legend/legend-pure`
(ground-truth every signature/semantic against these, never against memory).

## Standing tenets (violations get reverted)

1. **Java orchestrates, the database executes.** No host interpreter — value
   evaluation ALWAYS lowers to SQL. `Result`/`execute()` is a typing surface
   plus an orchestration handle, never a host object graph.
2. **One owner per behavior.** A shape is owned by the platform OR the
   harness, never both. Harness compensation for platform gaps is the
   cardinal sin (see the regex-parser and ExecHandle removals).
3. **Loud beats silently-different.** Unsupported shapes throw NAMED errors;
   a wrong-rows pass is worse than an honest ERROR.
4. **Exact FQN identification** — never `endsWith` matching.
5. **Features as coherent architecture,** grounded in the real engine's
   mechanism (read its .pure source), not calibrated to corpus pins.
   Audit 20a's not()-COALESCE finding is the canonical cautionary tale:
   the fix reproduced every pin but keyed at the wrong layer; the real
   mechanism was the pure `[0..1]` overload bodies inlining at the
   comparison site.

```
mvn -pl core test                # must be Failures: 0, Errors: 0, Skipped: 0
mvn -pl core install -DskipTests
cd engine && mvn -o test -Dtest='!RelationalCorpusRunner'   # 2729, 0 fail
mvn -o test -Dtest='RelationalCorpusRunner'                 # THE sweep
# diff the FAIL ledger vs the last committed baseline; classify EVERY delta
cd ../pct && mvn -o test                                    # 1109/1109
git commit (document deltas per-commit) && push origin main
```

- The FULL sweep is the only equivalence gate — family probes and solo
  URunner runs have lied (seed-context artifacts) multiple times.
- `docs/RELATIONAL_CORPUS.md` is GENERATED by the sweep. Never hand-merge
  it; regenerate after any merge.
- Known order-flapper: `testConcatenateClassAgg` (intra-group concatenation
  order) — flips between PASS/FAIL across sweeps; not a signal.
- Guardrails: files ≤3500 lines, methods ≤250 — split at the numbered-comment
  seams (GroupBySynthesis/NullSemantics/JoinChainEmission are the pattern).

## Hard-won lessons (do not relearn these)

- PCT pins 64-bit pure semantics (Long.MIN/MAX round-trips) — never trade
  semantics for golden text (the parseInteger INTEGER attempt).
- TypedSpec walkers with shadow-stops enter through `lambda.body()`, never
  the lambda node.
- The engine's corpus mappings often map association-shaped properties as
  class-typed Join PROPERTY mappings (`employees: @Firm_Person`), not
  AssociationMappings — route by what the mapping declares (the flatten
  lookup-unification, commit 5b6c0ca5).
- Scalar path flattens ride the projection FUNNEL (full demand machinery);
  the composed re-root serves only class-result shapes (commit 2307d6ff
  explains the 6-test regression that taught this).
- The `.rows` Typer marker exists for the K result frame's row-index vs
  envelope disambiguation; the resolver + lowerer erase it (audit 20c H1).
- Unbraced `let q = |expr;` lambda swallows following statements at parse —
  the corpus spells `{|...}`.
- Python edit scripts: assert EVERY anchor, write per-step (a late assert
  loses all earlier edits); zsh heredocs break on `#`/`—` in some quoting.
- Shell cwd resets between commands — `cd /Users/neema/legend/legend-lite`
  first, or use absolute paths / `mvn -f`.
